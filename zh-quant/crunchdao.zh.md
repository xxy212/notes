# WorldQuant 研讨会

## 规则

提交任务可能占用 30GB 内存。

每周运行时间为 5 小时。

## 数据

给定 276 个日期（这里称为天）的训练数据，每天约有 3000 只股票。
  * 起初每天约 500 只股票，后来增加到每天约 4000 只股票。
  * ❗️不能使用滞后的时间序列式特征！因为每天的行数不同。
  * 使用 `from sklearn.model_selection import TimeSeriesSplit` 按天划分数据。直接取连续块可能会不小心混合不同日期。

每只股票有 460 个特征。
  * 所有特征都被标准化为 Normal(0, 1)。

输出变量在 `(-1, 1)` 上均匀分布。

评价指标是在新日期上的 Spearman 秩相关系数。

## 提示

有些特征之间存在相关性。
  * 最小为 -0.49，最大（非自身）为 0.53。
  * 分布向负相关偏斜。
  * 可以用 Bonferroni 校正来检验显著性？

所有特征都是高斯分布——也许某些模型对此效果不错？

可以构建一张新表：对某一天的所有行计算每个特征的均值。这张表每天只有 1 行，因此可以使用滞后项。
  * 可能有助于描述每日市场环境。
  * 可以作为约束、中性化项或其他条件加入主预测。
  * 有些特征在 1 阶滞后上有显著的均值自相关。

可以用 RankNet、LambdaRank 或 LambdaMART 之类的技术直接对样本点排序。

差分特征包括：
  * 使用在当前日期样本之前日期训练的模型；
  * 今天的数据点到过去几天数据质心的距离；
  * 当前日期的样本在过去日期样本中的排名；
  * 当前日期样本到过去日期样本的最近邻。

特征重要性 / 特征剪枝。

特征工程，例如针对不同日期取滚动窗口，并从中提取均值、中位数和值域。

进一步的数据探索：
  * 我对相关矩阵比较好奇。

数据预处理，例如聚类或异常值检测，以降低预测偏差。

将损失函数微调成 MSE 之外的目标。

## 目前尝试过的策略

所有分数都是 Spearman 相关系数。

`rx` 表示每隔 x 天重新训练一次。

LASSO 线性模型，`alpha = 0.001`
  * 训练：3.538
  * 测试：r0 - 2.784，r2 - 2.958，r4 - 3.000，r6 - 2.979，r8 - 2.991

Elastic Net 线性模型
  * `alpha = 0`（LinearRegression）
    * 训练：3.591
  * `alpha = 0.000001`
    * 训练：3.608
  * `alpha = 0.0001`
    * 训练：3.748
    * 测试：r4 - 3.278
  * `alpha = 0.001`
    * 训练：3.734
    * 测试：r4 - 3.237
  * `alpha = 0.01`
    * 训练：2.309

使用优化参数的 LightGBM
  * 训练：2.676
  * 测试：r0 - 2.549，r2 - 2.844

使用基础参数的 XGBoost
  * 训练：0.851
  * 测试：r0 - 1.166

使用基础参数的 CatBoost
  * 训练：2.579
